# Сайт психолога-консультанта (Павел · КПТ)

Лендинг + блог психолога-консультанта со специализацией в когнитивно-поведенческой
терапии (КПТ). Сделан на **Next.js 15 (App Router) + TypeScript** со **статическим
экспортом** — это даёт максимальную скорость (важно для SEO) и возможность разместить
сайт на любом хостинге как обычную статику. Хостинг статики бесплатен или почти
бесплатен — расходов на инфраструктуру нет.

## Технологии

- **Next.js 15** (App Router), **React 19**, **TypeScript** (strict)
- Статический экспорт (`output: "export"`) → папка `out/`
- Шрифты через `next/font` (самохостинг, кириллица): Cormorant Garamond + Manrope
- Дизайн на CSS-переменных (без тяжёлых зависимостей)
- SEO: метаданные и Open Graph/Twitter на всех страницах, JSON-LD (Person,
  ProfessionalService, WebSite — на всех страницах; FAQPage — на главной; Article,
  Blog, BreadcrumbList — на страницах блога), `sitemap.xml` с честными датами
  обновления, `robots.txt`, `manifest`, фирменная страница 404
- Соответствие РФ: страница «Политика обработки ПДн» (152-ФЗ), cookie-баннер с явным
  согласием, Яндекс.Метрика подключается **только после** согласия, дисклеймер,
  без WhatsApp/Meta-сервисов
- ESLint 9 (`next/core-web-vitals` + `next/typescript`)

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000 — разработка
npm run lint     # проверка кода
npm run build    # сборка статики в папку out/
```

После `npm run build` всё, что нужно для хостинга, лежит в **`out/`**. Эту папку можно
залить на любой статический хостинг. Бюджетные варианты (заказов пока нет — платить
не за что):

- **Бесплатно:** GitHub Pages, Cloudflare Pages, Netlify (бесплатных лимитов хватит
  с огромным запасом).
- **РФ-хостинг** (если появится форма обратной связи — данные россиян должны храниться
  в РФ, 242-ФЗ): Timeweb / Beget / Reg.ru, тариф «статический сайт» — от ~100–200 ₽/мес.
  Домен `.ru` — ~200–400 ₽/год при регистрации.

## Архитектура

```
app/                 маршруты и SEO-файлы
  layout.tsx         общий каркас, метаданные, общесайтовый JSON-LD, аналитика
  page.tsx           главная (+ FAQPage JSON-LD)
  not-found.tsx      страница 404
  articles/          блог: список (+ Blog JSON-LD) и статьи (+ Article JSON-LD)
  privacy/           политика ПДн (152-ФЗ), noindex
  robots.ts / sitemap.ts / manifest.ts
components/          секции и UI-компоненты (Analytics — Метрика по согласию)
lib/
  types.ts           типы контента (структура данных проверяется компилятором)
  content.ts         ВСЕ тексты, контакты, цены, юр-данные, ID Метрики
  articles.ts        статьи блога (добавить статью = добавить объект в массив)
  jsonLd.ts          сборка структурированных данных (schema.org)
  utils.ts           общие утилиты (isSet, telHref, formatDate, …)
public/              статика (og.png, иконки, сканы дипломов)
документы/           юридические памятки (самозанятость, алименты, запуск)
```

Принципы: контент отделён от кода (правки текстов не требуют трогать компоненты);
структура контента типизирована (`lib/types.ts`); разметка schema.org собирается
только в `lib/jsonLd.ts`; страничная разметка (FAQ, статьи) выводится только на своих
страницах.

## Где менять контент

Почти всё — в двух файлах, код трогать не нужно:

- **`lib/content.ts`** — тексты всех секций, контакты, цены, юр-данные, ID Метрики.
- **`lib/articles.ts`** — статьи блога (добавить статью = добавить объект в массив;
  у статьи есть поле `updated` для честной даты обновления).

## ✅ Что заменить перед запуском (чек-лист)

В `lib/content.ts` → `siteConfig`:

| Поле | Сейчас | Заменить на |
|------|--------|-------------|
| `url` / `domain` | `https://example.ru` | ваш реальный домен |
| `telegram` | пусто | `https://t.me/ваш_username` |
| `max` | пусто | ссылка MAX (по желанию) |
| `vk`, `dzen` | пусто | ссылки на соцсети (по желанию) |
| `prices` + цены в `services` | «по запросу» | реальные цены |
| `inn` | `{{ИНН}}` | ИНН (обязателен на сайте по ст. 10 ЗоЗПП) |
| `yandexMetrikaId` | пусто | номер счётчика Метрики (подключится сам, после согласия) |
| `verification.yandex/google` | пусто | коды подтверждения Вебмастера / Search Console |
| `phone`, `email` | заполнены из резюме | проверьте |

Остальное:
- **Фото** — положите файл в `public/` (например `photo.jpg`), вставьте в `Hero`
  вместо плейсхолдера и укажите путь в `siteConfig.photo` (используется в разметке).
- **Дипломы/сертификаты** — сканы в `public/diplomas/`, путь и размеры (width/height)
  в `diplomas.items` в `lib/content.ts`.
- **OG-картинка** — замените `public/og.png` на изображение 1200×630 (лучше с фото).
  Имя файла менять не нужно.

## SEO после запуска (внешнее)

1. **Яндекс.Вебмастер** — добавить сайт (код подтверждения → `verification.yandex`),
   отправить `sitemap.xml`, указать регион (Санкт-Петербург).
2. **Google Search Console** — то же самое (код → `verification.google`).
3. **Яндекс.Метрика** — создать счётчик, номер → `yandexMetrikaId`; цели на клики
   по контактам. Счётчик загружается только после «Принять» в cookie-баннере.
4. **Яндекс.Бизнес** — карточка «Психолог» (Карты/Поиск).
5. Профили в каталогах психологов: **b17.ru**, **profi.ru**, psynavigator — со ссылкой
   на сайт (единые имя/контакты везде).
6. Telegram-канал/Дзен с экспертными статьями → ссылки на сайт (рост бренд-запросов).

## Юридический чек-лист (РФ, обновлён 02.07.2026)

Подробная памятка по самозанятости, алиментам и приёму оплаты —
в **`документы/самозанятость-и-сайт.md`**.

Уже сделано в коде:
- Политика обработки ПДн (`/privacy`) по 152-ФЗ; дисклеймер «консультирование не
  заменяет медпомощь».
- Cookie-баннер с **явным согласием** (Принять / Отклонить); Метрика грузится только
  после «Принять» (реализовано в `components/Analytics.tsx`).
- Юр-данные оператора в подвале (самозанятый / ИНН), контакты — по ст. 10 ЗоЗПП.
- Без сервисов Meta (WhatsApp / Instagram / Facebook). Учтите: с 01.09.2025 реклама
  в Instagram/Facebook запрещена законом (штрафы за каждый пост).

⚠️ Обязательно выполнить оператору (вне кода):
1. **Зарегистрироваться как самозанятый (НПД)** — приложение «Мой налог», 10–15 минут.
   На каждую оплату — чек из «Мой налог» (штраф за невыдачу: 20% суммы, повторно — 100%).
2. **Уведомить Роскомнадзор** об обработке ПДн (pd.rkn.gov.ru) — до запуска сбора
   заявок. Для самозанятого штраф за неуведомление 5–10 тыс. руб.
3. Указать **реальный ИНН** и статус **самозанятого** (`inn` в `lib/content.ts`).
4. **Локализация ПДн (242-ФЗ):** если добавите форму обратной связи — бэкенд/хостинг
   в РФ (Timeweb, Reg.ru, Selectel).
5. Реальные **отзывы** публиковать только с письменного согласия клиентов;
   демо-отзывы заменить (фейковые отзывы ФАС считает недостоверной рекламой).
6. Разместить **публичную оферту** (цена, отмена/перенос, «не медицинская услуга», 18+).

## ruflo

В проект добавлена интеграция **ruflo** (`.mcp.json`, `.claude/`). MCP-сервер ruflo
подключается при следующем запуске Claude Code; для работы агентов ruflo нужно задать
ключ AI-провайдера: `ruflo providers configure -p anthropic -k <ключ>`.
